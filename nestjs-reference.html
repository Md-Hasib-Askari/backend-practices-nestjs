Suggested stack

Every choice here is opinionated for a multi-tenant SaaS with auth, RBAC, and caching. Swap where it makes sense for your context.

Core

Framework
NestJS
Modular, DI-first, TypeScript native. Guards, interceptors, and pipes compose cleanly.
Core
Database
MongoDB + Mongoose
Flexible schema evolution. Per-tenant DB isolation is trivial with dynamic connections.
Core
Cache
Redis + @nestjs/cache-manager
Shared cache, session store, rate limit store, and pub/sub — one infrastructure tool.
Core
Auth
Passport.js + JWT + bcrypt
Proven, composable strategy pattern. LocalStrategy + JwtStrategy covers 90% of cases.
Core
Authorization
CASL
Attribute-based access control. Handles ownership, tenant isolation, and field-level permissions.
Core
Validation
class-validator + class-transformer
Decorative DTO validation. Pairs with NestJS ValidationPipe for automatic request validation.
Core
Config
@nestjs/config
Env-based config with validation via Joi or class-validator. registerAs() for namespaced config.
Core
Job queue
BullMQ + @nestjs/bull
Redis-backed job queue for async operations: email, export, provisioning, scheduled tasks.
Core

Optional / situational

API style
GraphQL (optional)
Add @nestjs/graphql + Apollo if clients need flexible queries. REST is simpler to start.
Optional
Email
Resend / Nodemailer
Resend for transactional email with great DX. Nodemailer for self-hosted / SMTP setups.
Optional
Storage
AWS S3 / Cloudflare R2
R2 is S3-compatible with no egress fees. Use for tenant data exports, file uploads.
Optional
Observability
Pino + OpenTelemetry
Pino for structured JSON logs. OTEL for distributed tracing across services.
Optional
Testing
Jest + Supertest
Jest for unit/integration, Supertest for e2e HTTP tests. NestJS testing module makes DI easy to mock.
Optional
Payments
Stripe
Stripe webhooks for subscription events. Store plan in the Tenant document, sync on webhook.
Optional

Project structure

A feature-based module structure. Each domain (auth, users, tenants, posts) is self-contained with its own controller, service, schema, repository, and guards.

Module template

// Every feature module follows this pattern:
@Module({
  imports: [
    MongooseModule.forFeature([{ name: Post.name, schema: PostSchema }]),
  ],
  controllers: [PostsController],
  providers: [PostsService, PostsRepository, CaslAbilityFactory],
  exports: [PostsService],
})
export class PostsModule {}

Useful commands

Project setup

# Bootstrap
npm i -g @nestjs/cli
nest new my-saas && cd my-saas

# Core dependencies
npm install @nestjs/mongoose mongoose
npm install @nestjs/passport passport passport-local passport-jwt
npm install @nestjs/jwt @nestjs/config @nestjs/cache-manager
npm install cache-manager cache-manager-redis-yet ioredis
npm install @nestjs/throttler helmet bcrypt class-validator class-transformer
npm install @casl/ability @casl/mongoose
npm install -D @types/passport-local @types/passport-jwt @types/bcrypt

NestJS CLI generators

# Generate a full feature module
nest g module posts
nest g controller posts
nest g service posts

# Or all at once with --no-spec
nest g resource posts --no-spec
